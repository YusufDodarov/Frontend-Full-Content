React.memo — Higher-Order Component аст, ки рендери компонентро мемоизатсия мекунад.
Он имкон медиҳад, ки: компонент дубора рендер нашавад, агар props тағйир наёфта бошанд.
Проблемаи аслӣ дар React:
Ҳар дафъа ки parent рендер мешавад
child низ рендер мешавад
ҳатто агар props-и child ҳамон бошад
Ин метавонад:
рендери зиёдатӣ
паст шудани performance
ба вуҷуд орад.
import { memo } from "react";

const Child = memo(function Child(props) {
  return <div>{props.value}</div>;
});
Чӣ кор мекунад memo:
React props-и қаблиро бо props-и нав муқоиса мекунад (shallow comparison)
Агар баробар бошанд → рендер намекунад
Агар фарқ дошта бошанд → рендер мекунад
Муҳим: memo ЧИРО муқоиса мекунад?
Танҳо props-ро.
❗ State-и дохилии компонент тағйир ёбад → рендер мешавад
❗ Context тағйир ёбад → рендер мешавад
Муносибат бо useCallback
Агар ба memo:
function ҳамчун prop фиристӣ
ва он function ҳар рендер нав эҷод шавад
➡️ memo фоида намедиҳад
Аз ҳамин сабаб:
memo + useCallback якҷо истифода мешаванд
